# HTTP状态码

### 状态码类别

| 状态码 |                        |            描述            |
| :----: | :--------------------: | :------------------------: |
|  1xx   | Informational信息响应  |     接受的请求正在处理     |
|  2xx   |    Success成功响应     |      请求正确处理完毕      |
|  3xx   |   Redirection重定向    | 需要进行附加操作以完成请求 |
|  4xx   | Client Error客户端错误 |     服务器无法处理请求     |
|  5xx   | Serve Error服务端错误  |     服务器处理请求出错     |

### 常见状态码

1. **2xx**

（1）200 OK

表示客户端发来的请求被服务端正确处理了。

（2）204 No Content

表示客户端发来的请求被服务端正确处理了但没有返回的内容。

（3）206 Partial Content

表示客户端进行了范围请求，服务端返回Content-Range指定的实体内容。

2. **3xx**

（1）301 Moved Permanently

永久重定向。表示请求的资源被分配了新的URI，以后应使用资源指定的URI。新的URI在响应头的Location字段指定，如果用户把旧的URI保存为书签，那么会按新的URI重新保存书签，同时搜索引擎在抓取内容时也把旧的网址替换为重定位后的网址。

（2）302 Found

临时重定向。与301相似只不过这是临时的，不会将新的URI保存为书签而是保留旧网页的URI，同样搜索引擎抓取新内容但不会把旧网址替换成新网址。

（3）304 Not Modified

浏览器缓存相关。该状态码表示客户端发送附带条件的请求时服务端允许请求访问资源但为满足条件。状态码304告诉客户端有缓存可以直接使用缓存中的数据，所以304状态码返回时响应报文的响应体是没有内容的。

（4）307 Temporary Redirect

也是临时重定向，和302的区别是302可以从POST变成GET（尽管302标准禁止这样做），而307不会从POST变成GET。

3. **4xx**

（1）400 Bad Request

表示请求报文中存在语法错误，需要修改请求后再次发送。

（2）401 Unauthorized

表示请求需要有通过HTTP认证的认证信息。若之前已进行过一次请求则表示用户认证失败。

返回含有401状态码的响应头必须包含`WWW-Authenticate`字段用于向客户端发起认证请求。

（3）403 Forbidden

表示客户端的请求被服务器拒绝了（访问权限出现问题）。

（4）404 Not Found

表示访问的资源不存在，服务器无法找到请求的资源。

（5）405 Method Not Allowed

表示客户端请求的方法能够被识别但服务端不允许使用该方法。

4. **5xx**

（1）500 Internal Serve Error

表示服务器在执行请求时发生了错误。

（2）501Not Implemented

表示服务器不支持请求中的某个功能。与405不同的是405表示能够识别该方法但禁用，501是指服务器不支持这个方法。

（3）503 Service Unavailable

表示服务器暂时处于超负荷或正在停机维护。
